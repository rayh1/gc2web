# ADR-0004: Normalize generated metadata for browse and search result cards

Date: 2026-06-25
Status: Accepted

## Context

Search and browse results currently expose only titles and IDs, while the PRD requires clearer hierarchy and richer contextual cues for comparing similar people. The content schema presently guarantees only `title`, so the needed context must be made explicit.

## Decision

Extend generated entity frontmatter and the Astro content schema with normalized metadata for result cards, such as lifespan, place, branch, and relationship summary, and use that metadata in a unified browse/search card layout.

## Consequences

- Search and browse can show disambiguating context consistently when the data exists.

* The generator contract and search/indexing code must evolve together, and sparse records need graceful fallbacks.
