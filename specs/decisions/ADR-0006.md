# ADR-0006: Cover the redesign with browser smoke and component-level tests

Date: 2026-06-25
Status: Accepted

## Context

The redesign changes navigation, homepage composition, search/browse cards, and entity-page structure. The repository has no existing frontend test harness, but the user explicitly chose both browser smoke coverage and component-level tests for this work.

## Decision

Introduce a browser-level smoke suite for key archive flows and a component-level test layer for condition-heavy UI pieces such as navigation, search, and result cards.

## Consequences

- The redesign gains automated regression coverage for both real page flows and smaller UI behaviors.

* The effort must add frontend test tooling and keep those tests updated alongside the new UI contract.
