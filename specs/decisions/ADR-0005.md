# ADR-0005: Keep sticky navigation and add breadcrumb-style page context

Date: 2026-06-25
Status: Accepted

## Context

The site already has a sticky header and mobile menu, but the PRD calls for clearer orientation across desktop and mobile views, especially when page structure changes. Active-link styling alone does not communicate current-page context strongly enough.

## Decision

Retain the sticky header and mobile drawer, and add breadcrumb-style current-context cues for entity, browse, and search pages.

## Consequences

- Visitors keep global navigation while gaining an explicit sense of where they are in the archive.

* Shared layouts and page entry points must pass breadcrumb context deliberately instead of relying only on static links.
