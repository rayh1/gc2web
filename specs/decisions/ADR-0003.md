# ADR-0003: Use generator-backed grouping hooks for entity pages

Date: 2026-06-25
Status: Accepted

## Context

Entity pages are rendered from generated markdown and must handle both sparse and dense biographies while preserving ASCII trees, citations, and genealogical content structure. CSS-only prose restyling is not reliable enough to provide the grouping and density handling required by the PRD.

## Decision

Allow the generation pipeline to add normalized grouping hooks and, if necessary, restructure emitted entity-page markup while keeping the underlying genealogy source model unchanged.

## Consequences

- Entity pages can present identity, relationships, chronology, and source material as stable regions instead of one long prose stream.

* The redesign now depends on coordination between Astro templates and `gen_site/gen_site.py` rather than frontend-only edits.
